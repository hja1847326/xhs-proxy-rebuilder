# Validation Matrix V1

## 目标

把这套可复制方案按真实资源场景拆成验证矩阵，明确：
- 输入长什么样
- 预期生成多少代理
- 当前验证到什么程度
- 还缺什么真实样本

---

## 场景矩阵

| 场景 ID | 资源结构 | 输入来源 | 预期代理数 | 当前状态 | 备注 |
|---|---|---:|---:|---|---|
| S1 | 主 IP | inventory / profile | 1 | 逻辑可支持，未单独样例化 | 最小场景 |
| S2 | 主 IP + 2 VIP | profile 默认值 | 3 | 逻辑可支持，未单独真实样例化 | 适合最小矩阵起步 |
| S3 | 固定基础 4 路（主 IP + 2 VIP + 1 固定辅助网卡） | profile 默认值 | 4 | **已验证生成链路** | 当前基础盘 |
| S4 | 固定基础 4 路 + 1 条 txt 扩展网卡 | `1_ip_origin...` 实际样本 | 5 | **已验证到运行态 netns 扩展链路** | 第 5 路按 `VLAN + netns + gost` 路线成立 |
| S5 | 固定基础 4 路 + N 条 txt 扩展网卡 | ip_origin 多行 | 4+N | 部分逻辑已支持，缺足量真实样本 | 主体扩展路径 |
| S5 | 10IP 级别组合 | `huaweicloud-10ip` profile | 取决于资源数 | **profile 已有，缺足量真实样本** | 当前只是策略预设 |
| S6 | 20IP 级别组合 | `huaweicloud-20ip` profile | 取决于资源数 | **profile 已有，缺足量真实样本** | 当前只是策略预设 |

## 已验证内容

### S3：主 IP + 2 VIP + 1 辅助网卡

#### 输入
- 主 IP：`192.168.0.10`
- VIP：`192.168.0.11`、`192.168.0.12`
- 辅助网卡样本：
  - `192.168.0.88 1866 fa:16:3e:8b:e3:3b`

#### 当前结论
- `ip_origin` 解析成功
- inventory 生成成功
- lint 成功
- xray config 生成成功
- proxies 导出成功
- manifest / bundle / release 链路成功
- 在真实新机上已确认：第 5 路不能用主命名空间旁挂模型，而应使用 `VLAN + netns + gost`
- 新机 `ns1866` 运行态验证已拿到正确公网出口：`122.9.187.215`

#### 当前限制
- 只有 1 条真实辅助网卡样本
- 还不足以代表 10IP / 20IP 场景

---

## 当前可视为“功能已具备但样本不足”的场景

### S4：主 IP + 2 VIP + 多辅助网卡
当前工具链已经具备：
- 多行 `ip_origin` 解析
- 递增端口
- 递增账号生成
- inventory / build / release 输出

但还缺：
- 多条真实 `ip_origin` 样本
- 不同 VLAN / MAC 组合的真实输入
- 更高代理数量下的实测结果

### S5 / S6：10IP / 20IP
当前已具备：
- profile 预设
- 随机密码策略
- release / bundle / preflight / remote SOP

但目前还只是：
- **结构上支持**
- **文档上支持**
- **策略上支持**

还没达到：
- **多资源真实样本验证完毕**

---

## 推荐验证优先级

### P1：补多行 `ip_origin` 真实样本
目标：至少拿到 3~5 条真实辅助网卡记录，验证 S4。

### P2：补 10IP 真实资源清单
目标：验证 `huaweicloud-10ip.yaml` 不只是概念 profile。

### P3：补 20IP 真实资源清单
目标：验证更高代理数时的构建和部署可行性。

---

## 当前一句话结论

> 这套工具链已经对 **S3（主 IP + 2 VIP + 1 辅助网卡）** 形成了较完整的真实样本链路验证；对 **S4/S5/S6** 已经具备结构与策略能力，但还缺足量真实华为云资源样本来证明“通用性”而不是“只在小样本上成立”。
